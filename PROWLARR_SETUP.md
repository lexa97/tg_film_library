# Настройка Prowlarr для поиска торрентов

Prowlarr — это менеджер индексаторов торрент-трекеров, который обеспечивает единый API для поиска раздач по множеству источников.

## Быстрый старт с Docker Compose

Prowlarr уже включен в `docker-compose.yml` и будет автоматически запущен вместе с ботом.

### Шаг 1: Первый запуск

```bash
docker-compose up -d
```

После запуска Prowlarr будет доступен по адресу: **http://localhost:9696**

### Шаг 2: Первоначальная настройка Prowlarr

1. Откройте http://localhost:9696 в браузере
2. При первом запуске Prowlarr предложит пройти настройку:
   - Выберите язык интерфейса (можно выбрать русский)
   - Настройте аутентификацию (опционально, но рекомендуется)

### Шаг 3: Получение API ключа

1. Перейдите в **Settings → General** (Настройки → Общие)
2. В разделе **Security** найдите поле **API Key**
3. Скопируйте значение API ключа
4. Добавьте его в файл `.env`:

```env
PROWLARR_API_KEY=ваш_скопированный_api_ключ
```

### Шаг 4: Добавление индексаторов (трекеров)

Prowlarr поддерживает множество публичных и приватных трекеров.

#### Добавление публичного индексатора:

1. Перейдите в **Indexers** → **Add Indexer** (Индексаторы → Добавить индексатор)
2. Найдите нужный трекер в списке (например, RuTor, RuTracker и т.д.)
3. Нажмите на него и заполните настройки:
   - **Name**: название (можно оставить по умолчанию)
   - **Enable**: включено (должна быть галочка)
   - Остальные параметры обычно можно оставить по умолчанию
4. Нажмите **Test** для проверки
5. Нажмите **Save**

#### Добавление приватного индексатора:

Для приватных трекеров (например, RuTracker с авторизацией):

1. Добавьте индексатор как описано выше
2. Заполните дополнительные поля:
   - **Username**: ваш логин на трекере
   - **Password**: ваш пароль на трекере
   - **Cookie**: если требуется (можно получить из браузера)
3. Нажмите **Test** и **Save**

#### Рекомендуемые индексаторы для русскоязычного контента:

- **RuTor** (публичный)
- **RuTracker** (требует регистрации)
- **Kinozal** (требует регистрации)
- **Torrents.csv** (публичный)

### Шаг 5: Перезапуск бота

После настройки Prowlarr и добавления API ключа в `.env`, перезапустите бота:

```bash
docker-compose restart bot
```

### Шаг 6: Проверка работы

1. Откройте бота в Telegram
2. Найдите любой фильм через поиск
3. Нажмите кнопку **🧲 Magnet** под описанием фильма
4. Бот должен показать список доступных раздач с информацией о качестве, размере и количестве сидов

## Устранение проблем

### Prowlarr не находит раздачи

**Возможные причины:**

1. **Не добавлены индексаторы**
   - Проверьте, что в Prowlarr добавлен хотя бы один активный индексатор

2. **Индексаторы отключены**
   - В Prowlarr → Indexers убедитесь, что галочка "Enable" стоит у индексаторов

3. **Неправильный API ключ**
   - Проверьте, что API ключ в `.env` совпадает с ключом в Prowlarr → Settings → General

4. **Prowlarr недоступен**
   - Проверьте, что контейнер запущен: `docker ps | grep prowlarr`
   - Проверьте логи: `docker-compose logs prowlarr`

### Ошибка подключения к Prowlarr

**Если бот не может подключиться к Prowlarr:**

1. Убедитесь, что в `.env` указан правильный URL:
   ```env
   PROWLARR_URL=http://prowlarr:9696
   ```
   (для Docker Compose используется имя сервиса `prowlarr`)

2. Проверьте, что оба контейнера (bot и prowlarr) в одной сети:
   ```bash
   docker network inspect tg_film_library_film_library_network
   ```

### Фильтр по качеству не работает

Бот настроен на поиск только раздач качества **1080p и выше**. Если раздачи не находятся:

1. Попробуйте другие индексаторы
2. Убедитесь, что фильм доступен в HD качестве
3. Проверьте, что название фильма и год корректны

## Дополнительные настройки Prowlarr

### Настройка прокси (если нужен)

Если некоторые трекеры заблокированы в вашей стране:

1. Перейдите в **Settings → General**
2. Найдите раздел **Proxy**
3. Укажите параметры вашего прокси-сервера
4. Сохраните настройки

### Автоматическая синхронизация

Prowlarr может автоматически синхронизировать индексаторы с другими приложениями *arr (Sonarr, Radarr и т.д.), но для нашего бота это не требуется.

## Безопасность

### Рекомендации:

1. **Включите аутентификацию** в Prowlarr (Settings → General → Security)
2. **Не публикуйте API ключ** в открытом доступе
3. **Используйте сложные пароли** для приватных трекеров
4. **Регулярно обновляйте** Prowlarr до последней версии

## Обновление Prowlarr

Для обновления Prowlarr до последней версии:

```bash
docker-compose pull prowlarr
docker-compose up -d prowlarr
```
