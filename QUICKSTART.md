# Быстрый старт

## Запуск через Docker Compose (рекомендуется)

### 1. Проверьте `.env` файл

Убедитесь, что файл `.env` содержит корректные токены:

```env
BOT_TOKEN=dfdfsdfs
TMDB_API_KEY=eyJhbGci...ваш_ключ...
DATABASE_URL=postgresql+asyncpg://tgfilm:tgfilm_secret@10.1.20.24:15432/tg_film_library
```

### 2. Запустите проект

```bash
docker-compose up -d
```

### 3. Проверьте логи

```bash
docker-compose logs -f bot
```

### 4. Остановка

```bash
docker-compose down
```

## Локальный запуск (для разработки)

### 1. Установите зависимости

```bash
# Только для работы бота
pip install -r requirements.txt

# С тестами и линтером (рекомендуется для разработки)
pip install -r requirements-dev.txt
```

### 2. Настройте БД

Убедитесь, что PostgreSQL запущен и доступен по адресу из `.env`.

### 3. Инициализируйте БД

```bash
python initdb.py
```

Скрипт автоматически создаст все необходимые таблицы.

### 4. Запустите бота

```bash
python -m app.main
```

## Использование Makefile (Linux/Mac)

```bash
make install   # Установка зависимостей
make test      # Запуск тестов
make lint      # Проверка кода
make format    # Форматирование
make migrate   # Применение миграций
make up        # Запуск Docker Compose
make down      # Остановка Docker Compose
make clean     # Очистка временных файлов
```

## Разработка

### Добавление новых функций

1. Создайте ветку
2. Внесите изменения
3. Запустите тесты: `pytest`
4. Проверьте код: `ruff check .`
5. Создайте pull request

### Изменение схемы БД

После изменения моделей в `app/db/models.py`:

1. Остановите бота
2. Для создания новых таблиц — просто перезапустите, `initdb.py` создаст их автоматически
3. **Важно:** Скрипт не изменяет существующие таблицы

Для полного пересоздания БД:
```bash
docker-compose down -v  # Удаляет том с БД
docker-compose up -d    # Создает чистую БД
```
