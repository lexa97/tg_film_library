# 🚀 Быстрый старт Prowlarr Integration

## Что нужно сделать СЕЙЧАС

### 1️⃣ Обновите .env файл

Откройте файл `.env` и добавьте следующие строки:

```env
PROWLARR_URL=http://prowlarr:9696
PROWLARR_API_KEY=заполните_после_первого_запуска
```

Сохраните файл.

### 2️⃣ Запустите Docker Compose

```bash
docker-compose up -d
```

Это запустит 3 контейнера:
- ✅ PostgreSQL (база данных)
- ✅ **Prowlarr** (новый сервис для поиска торрентов)
- ✅ Bot (ваш Telegram бот)

### 3️⃣ Настройте Prowlarr (первый раз)

1. Откройте в браузере: **http://localhost:9696**

2. Пройдите первоначальную настройку:
   - Выберите язык (можно русский)
   - Настройте аутентификацию (по желанию)

3. **Получите API ключ:**
   - Перейдите: **Settings → General**
   - Найдите секцию **Security**
   - Скопируйте значение поля **API Key**

4. **Добавьте API ключ в .env:**
   ```env
   PROWLARR_API_KEY=ваш_скопированный_api_ключ
   ```

5. **Добавьте индексаторы (трекеры):**
   - Перейдите: **Indexers → Add Indexer**
   - Найдите и добавьте трекеры (например):
     - **RuTor** (публичный)
     - **RuTracker** (требует регистрацию)
     - **Kinozal** (требует регистрацию)
   - Для каждого нажмите **Test** → **Save**

6. **Перезапустите бота:**
   ```bash
   docker-compose restart bot
   ```

### 4️⃣ Проверьте работу (необязательно)

Запустите тестовый скрипт:

```bash
python test_prowlarr.py
```

Он проверит подключение к Prowlarr и выполнит тестовый поиск.

### 5️⃣ Попробуйте в Telegram!

1. Откройте вашего бота в Telegram
2. Найдите любой фильм (например: "Интерстеллар")
3. Нажмите кнопку **📥 Скачать** под описанием
4. Выберите раздачу из списка
5. Раздача автоматически отправится в ваш торрент-клиент!
6. Проверьте торрент-клиент - загрузка должна начаться

## 🆘 Проблемы?

### Раздачи не находятся

1. Проверьте, что в Prowlarr добавлены индексаторы: http://localhost:9696
2. Убедитесь, что индексаторы включены (галочка "Enable")
3. Попробуйте другой фильм

### Ошибка "Prowlarr недоступен"

1. Проверьте, что контейнер запущен: `docker ps | grep prowlarr`
2. Проверьте логи: `docker-compose logs prowlarr`
3. Убедитесь, что API ключ правильный

### Другие проблемы

См. раздел "Устранение проблем" в `PROWLARR_SETUP.md`
